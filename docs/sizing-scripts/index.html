Wiz Sizing Scripts

Field-hardened Wiz resource-discovery and sizing scripts for estimating billable units before connectors are enabled. All scripts have local improvements for reliability and operability; the scripts live in the repository and gists are kept as instruction pages.

sizing resource discovery python powershell

Overview

The scripts estimate billable units before Wiz has live connector data. After a connector is enabled, the Wiz Settings > Licenses page is the better source of truth.

Code

Counts active developers across VCS and CI/CD systems using commits and activity from the last 90 days.

Counts compute, data, registry, and infrastructure resources across cloud and infrastructure providers.

Microsoft 365 Canonical Path

The Microsoft 365 sizing script is organized with the rest of the sizing scripts. Use this path for new runbooks and customer instructions.

Invoke-WebRequest -Uri https://raw.githubusercontent.com/adilio/wiz-tools/main/sizing-scripts/saas/microsoft-365/365_Sizing_Script.ps1 -OutFile 365_Sizing_Script.ps1
./365_Sizing_Script.ps1

The older published m365-sizing-xl URL is retained only as a compatibility documentation page for links that were already shared. It points to the same canonical script path above.

Common Flags

  • --all scans all accessible accounts, projects, or subscriptions where supported.
  • --data includes buckets and databases where supported.
  • --images includes registry container images where supported.
  • --output-dir writes all CSV and log output to the specified directory (all scripts).
  • --max-run-minutes stops scanning after N minutes and writes partial results (cloud scripts).
  • --checkpoint-interval writes partial output every N accounts/subscriptions/compartments (cloud scripts).

What Changed — All Scripts

All scripts keep the original sizing intent. Every script received a common set of hardening improvements; cloud scripts with org-scale account lists also received scoping and resume flags. Script-specific correctness fixes are listed below.

Universal Improvements (all 14 scripts)

ImprovementWhat changedWhy it changed
Elapsed-time progressAdds +MM:SS [SCAN] / [DONE] status lines around each work unit. Code scripts add --progress-interval.Operators cannot distinguish a slow scan from a hung one.
Partial output on Ctrl-C / failureSignal handler writes partial results before exiting. Main loop wrapped in try/except to write partial results before re-raising.A multi-hour scan interrupted at hour 4 previously wrote nothing.
--output-dirAll file writes routed through output_path(); os.makedirs(exist_ok=True) on first write.Organizes output for pilots and parallel scans against multiple tenants.
Worker-thread exception loggingThreaded scripts log each failed worker with account/task name and exception text instead of silently counting it.The old pattern hid all thread-level bugs — the primary silent-undercount mechanism.

Org-scale cloud scripts — scoping and resume flags

AWS (both), Azure, OCI, Alibaba, and Snowflake now accept: --max-run-minutes, --max-accounts / --max-subscriptions / --max-compartments, --checkpoint-interval, --start-after-account (or subscription/compartment), --include-*-regex, --exclude-*-regex.

AWS Standard — correctness fixes

FixWhy it mattered
DocumentDB pagination called describe_db_instances on page 2+ then read DBClusters → KeyErrorSilent undercount in any account with more than 100 DocumentDB clusters.
Lightsail instances checked PlatformDetails (no such key); real key is platformVirtual Machine Sensors were always zero for Lightsail Linux instances.
EKS Fargate while-loop was dead code; if activated naively it would double-count podsReplaced with a single-pass profile-existence check to prevent a latent double-count.
Thread-unsafe totals[...] += n from worker threadsDict increment is not atomic; concurrent writes produce occasional miscounts.

Azure — correctness fixes

FixWhy it mattered
AKS summed only agent_pool_profiles[0] (both SDK and --graph paths)Multi-nodepool clusters were undercounted for Container Hosts and Kubernetes Sensors.
scale_sets / registries unbound when the list call raised → silent NameError in worker threadPrevented any count from a subscription where a list call failed.
No per-request timeout on SDK clients; added --request-timeout via socket.setdefaulttimeout()A hung socket stalled an entire subscription indefinitely.

OCI — correctness fixes

FixWhy it mattered
Shared config dict aliased (not copied) across parallel region threads writing config['region']Scans ran against whichever region the last thread wrote — silent miscounts across regions.
Oracle-Tags → CreatedBy direct index killed the region thread on any instance missing that tagAny instance without the tag dropped the entire region silently.
One get_image API call per instance; added module-level image OS cacheFleets sharing a handful of images made thousands of redundant API calls.
Added retry_strategy=oci.retry.DEFAULT_RETRY_STRATEGY to all six client constructionsSDK default is no retries; transient errors were silently swallowed.

Alibaba Cloud — correctness fixes

FixWhy it mattered
ACK pagination: floor instead of ceil, < instead of <=, request.page_number(n) call instead of = n assignment → TypeError after page 1Any tenant with more than one page of ACK clusters silently undercounted Container Hosts and Kubernetes Sensors.

Snowflake — correctness fixes

FixWhy it mattered
split('-') failed for accounts with hyphens in org/account name; validation boolean was invertedCommon MYORG-MY-ACCOUNT format failed at startup.
Database name interpolated unquoted into INFORMATION_SCHEMA queryDatabases with lowercase or special-character names were silently skipped.
One connection opened per database call; now one connection per account with login_timeout / network_timeoutConnection overhead multiplied with database count; connections were unclosed on error.

GitHub — correctness fixes

FixWhy it mattered
Rate-limit retry called datetime.fromtimestamp() on the module instead of datetime.datetime.fromtimestamp(int(header))Retry decorator crashed on every rate limit hit; failure was swallowed → silent undercount on large orgs.
90-day since= window used naive local time instead of UTCCommits near the boundary were incorrectly included or excluded in non-UTC timezones.
if not repository: print(...) without sys.exit(1) → continued to iterate [None]Bad repository reference produced a confusing crash instead of a clean exit.

GitLab — correctness fixes

FixWhy it mattered
output_results() referenced args.pro (typo) → AttributeError on every --proj runEvery project-scoped run lost its console summary and cross-VCS rollup after scanning.
if not group/project: print(...) without sys.exit(1) → None.projects crashBad group or project name produced a confusing crash instead of a clean exit.

HCP Terraform — correctness fixes

FixWhy it mattered
Paginated API call looped forever on any non-200/429 response; added capped retries with backoffA 401/403/500 hung the script indefinitely while spamming errors.
configuration_version['id'] on {} → KeyError terminated the whole scan with no outputAny workspace with an unavailable config version ended the run before writing results.

PaginationNo operator-visible paging controlsAdds --project-page-size and --commit-page-sizeImproves large-org behavior and makes API work visible
